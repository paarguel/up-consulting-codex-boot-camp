# Teacher Notes 01 - AI History Whiteboard

## Goal

Give a short, confidence-building history of AI tools so Codex feels like an evolution, not a random new app.

## Timing

10 to 15 minutes.

## Whiteboard

```text
Search engine -> ChatGPT -> Custom GPTs and connectors -> Codex
```

## Talk track

### 1. Search engines

Search engines helped us find information, but the human still had to open pages, compare sources, read everything, and create the final artifact.

Use example:

> If you need to plan a youth night, a search engine can help you find ideas. It does not read your room notes, volunteer list, budget, and old email draft together.

### 2. ChatGPT

ChatGPT made AI conversational.

It can explain, summarize, brainstorm, rewrite, and help you think. But it usually needs you to paste or describe the context.

Use example:

> ChatGPT can draft a parent email if you give it the event details. If you leave out details, it may make assumptions.

### 3. Custom GPTs, plugins, and connectors

These made assistants more specialized.

They can have standing instructions or connect to specific tools and data sources when authorized.

Use example:

> A custom assistant might know your tone guide or help with a specific recurring job.

### 4. Codex

Codex is an agentic workbench around a folder.

It can read files, organize work, create outputs, explain changes, and help save reusable workflows.

For this boot camp, the project folder is the learner's desk:

```text
inputs -> outputs -> workflows
```

## Key sentence

> We are moving from asking AI one question at a time to giving it a workbench, a goal, boundaries, and a review process.

## What not to over-teach

Do not spend too long on technical model details.

The learner needs a usable mental model:

```text
AI is useful at drafting and organizing.
AI still needs human review.
Codex works best when the folder gives it context.
```
