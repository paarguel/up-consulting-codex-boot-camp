# From Office Pain to Codex Prompt

Use this handout when the learner says, "I do not know what to ask Codex to do."

## Translation table

| Human frustration | Codex task |
|---|---|
| This is scattered. | Organize these inputs into a clean brief. |
| I keep rewriting this. | Create reusable versions for different channels. |
| I do not trust this list. | Find duplicates, missing fields, and exceptions. |
| I need to update people. | Draft a status report and follow-up email. |
| I need to remember the process. | Turn this into an SOP and checklist. |
| I have to decide between options. | Compare against criteria and flag tradeoffs. |
| I am drowning in messages. | Triage into urgent, waiting, FYI, and follow-up. |

## Workflow canvas

Fill this in before prompting:

```text
Pain:

Inputs I have:

Output I want:

Audience:

Tone:

What Codex must not invent:

Review risks:

Definition of done:
```

## Starter prompt

```text
I want to turn this office frustration into a Codex workflow.

Pain:
[describe the annoying task]

Inputs I have:
[list files, notes, emails, spreadsheets, or drafts]

Output I want:
[brief, email, report, tracker, checklist, SOP, comparison]

Review risks:
[privacy, dates, money, names, approvals, final sending]

First, help me turn this into a clear workflow prompt. Do not create the final output yet.
```
